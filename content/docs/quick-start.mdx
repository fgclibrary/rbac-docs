前端权限框架所提供的功能是高聚合的，这意味着如果您需要快速让系统具备基础的前端权限功能，直接引入框架模块，做简单配置，系统即可具备：用户管理、角色管理、组织管理、权限管理（角色与权限组关联）的能力。

### 导入框架工程 [step]

在您活字格应用的工程文件里，单击数据页签，选择**活字格文件**。

![导入框架工程文件](/images/installation-export.png)

在导入引导窗口勾选如下模块：

| 分类    | 选择项                 |
|-------|---------------------|
| 页面    | 账户管理、角色管理、组织管理、权限管理 |
| 母版页   | FGC_母版页             |
| 组件    | 全部组件                |
| 数据表   | 全部数据表               |
| 服务端命令 | 全部服务端命令             |
| 单元格样式 | 全部单元格样式             |
| 表格样式  | 全部表格样式              |
| 图片    | 除内置图片之外的其他图片        |
| 其他    | 附件                  |

### 视图调整 [step]

活字格的用户数据均通过内建视图进行呈现，为确保框架能正常运行，我们需要对内建视图增加一些辅助字段。

1. 角色分类关系视图

  - 添加公式字段 `角色唯一标识`：
  ```txt
    [Id]&"~"&[是否角色]
  ```

  - 添加公式字段 `父角色唯一标识`：
  ```txt
    IF(ISBLANK([父节点Id]),"",[父节点Id]&"~0")
  ```

  - 添加公式字段 `图标ID`：
  ```txt
    IF([是否角色]=1,2,1)
  ```
  将该公式字段与 `角色树形控件图标表`的 `ID` 设置为关联字段。

2. 组织信息视图

  - 添加公式字段 `图标`：
  ```txt
    1
  ```

### 视图关联 [step]

将上一步设置的与图标相关的公式字段和第一步导入的图标表进行关联：

| 字段     | 所属视图     | 目标表与字段             |
|--------|----------|--------------------|
| `图标ID` | 角色分类关系视图 | `角色树形控件图标表` 的 `ID` |
| `图标`   | 组织信息视图   | `组织树形控件图标表` 的 `ID` |

### 🎉 已完成 [step]

点击立即运行！现在您就可以在浏览器中访问权限管理模块，并对系统中的用户、角色、组织结构以及权限进行管理。让我们看看框架背后的技术设计。
